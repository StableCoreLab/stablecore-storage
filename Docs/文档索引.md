# Storage 文档索引

> 状态：持续维护
> 责任范围：`Storage/Docs/` 文档治理
> 适用范围：存储模块自身的架构、设计、开发维护文档

本文档只记录存储模块自身的当前有效文档，并作为 `Storage/Docs/` 的正式索引。

产品功能设计文档不纳入本索引；该类文档仍归 `Quantity/Docs/` 统一管理。

## 1. 索引原则

1. 仅收录存储模块自身文档，不收录产品功能设计文档。
2. 文档按 `基础 / 实现 / 专题 / 确认单` 四类组织。
3. 同一主题若存在总览文档和专题文档，优先列当前主文档。
4. 规划、评审、改进和待细化内容可单独维护，但不与产品文档混排。
