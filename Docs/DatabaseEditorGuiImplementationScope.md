# 数据库编辑器 GUI 当前实现范围

本文说明 Tools/DatabaseEditor 当前已经完成的能力，以及仍保留在后续迭代中的内容。

## 已完成

### 1. 基础数据库工作流

- 新建 SQLite 数据库。
- 打开已有 SQLite 数据库。
- 单窗口内切换与浏览当前数据库中的表。
- 通过列表选择当前表。

### 2. 原始记录编辑

- 创建表。
- 为当前表新增列。
- 新增记录。
- 删除记录。
- 在表格中编辑可编辑的事实列。

### 3. 关系列选择器

- 对关系列提供专门的候选记录选择器。
- 支持按标签、记录 id 和预览列过滤。
- 能把选中记录写回当前单元格。

### 4. 数据网格体验增强

- 支持基础排序。
- 支持全文本过滤。
- 支持表摘要显示。
- 支持 Schema 检查器与记录检查器联动。

### 5. 会话级计算列

- 新增会话级计算列。
- 编辑会话级计算列。
- 删除会话级计算列。
- 支持普通字段与会话级计算列之间的显式互转，转换时保持同名，并在普通字段转计算列时清理原字段数据。
- 在右侧面板查看当前表的会话计算列定义。

## 当前约束

当前实现仍存在以下约束：

- 计算列仅为会话级，不会持久化到数据库。
- 过滤为全文本匹配，不支持高级条件构造。
- 诊断视图目前以摘要文本为主，不是完整工作台。
- Schema 编辑能力以“新增列”为主，尚未覆盖复杂重构。
- 更严格的权限、防误删、防误改保护仍待补充。

## 后续可继续增强

建议的后续方向：

- 增加更完整的 Schema 修改能力。
- 支持导入导出计算列配置。
- 增加批量编辑、批量导入与差异预览。
- 增加更强的启动诊断和恢复可视化能力。
- 面向产品化继续优化 UI 反馈、可用性与保护机制。
## 当前补充

- 选中 Schema 字段后已可通过 `Edit Column...` 原地更新同名字段定义。
- 工具栏与菜单已补充 `Create Backup Copy...` 入口，用于调用底层备份复制能力。
