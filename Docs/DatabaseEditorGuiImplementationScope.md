# 数据库编辑器 GUI 当前实现范围

本文档说明 `Tools/DatabaseEditor` 当前已经完成的能力，以及仍然保留在后续迭代中的内容。

## 已完成

### 1. 基础数据库工作流

- 新建 SQLite 数据库
- 打开已有 SQLite 数据库
- 一库一窗口
- 左侧表列表浏览
- 创建表
- 添加事实列和关系列

### 2. 原始记录编辑

- 表格方式浏览记录
- 新增记录
- 删除选中记录
- 直接编辑事实列原始值
- 每次用户编辑动作都单独提交，并形成一条 Undo 日志
- `Undo / Redo`

### 3. 关系字段选择器

- 选中关系字段单元格后，可通过工具栏、菜单或顶部按钮打开关系选择器
- 关系选择器会读取目标表全部可用记录
- 优先使用 `Name / Title / Code / 编号 / 名称` 作为候选显示标签
- 同时显示 `RecordId` 和预览字段，避免用户手工输入 `recordId`

### 4. 数据网格体验增强

- 顶部快速过滤框
- 按当前过滤条件实时刷新表格
- 启用列排序
- 启用交替行背景
- 显示当前表名、过滤后记录数、总记录数、当前选中记录摘要
- 保留 Inspector 和 Diagnostics 联动刷新

### 5. 会话级计算列

- 支持在当前表视图上新增“仅当前工具会话有效”的计算列
- 支持三类定义：
  - `Expression`
  - `Rule`
  - `Aggregate`
- 支持设置：
  - `name`
  - `displayName`
  - `valueKind`
  - `cacheable`
  - 依赖字段
  - 聚合关系和聚合字段
- 新增的会话级计算列会显示在数据网格中
- 右侧新增“Session Computed Columns”面板用于查看当前表已挂载的计算列
- 支持基于当前表内唯一 `name` 编辑和删除会话级计算列
- 编辑或删除后会重建当前表的会话计算列视图并刷新网格
- 会话级计算列不会写回数据库，关闭编辑器后失效

## 当前约束

- 关系字段仍然只支持单值引用，不支持多选关系
- 会话级计算列的新增、编辑、删除不进入数据库 `Undo / Redo`
- 计算列面板不支持导入导出
- `Rule` 计算列依赖底层 `ruleId` 注册表；GUI 本身不负责提供规则实现
- 过滤目前是全文本匹配，不包含高级条件构造
- 排序和过滤作用于当前表格视图，不会回写数据库

## 后续可继续增强

- 关系字段单元格专用 delegate，下拉即选而不是通过独立对话框
- 会话级计算列的复制
- 更强的筛选器，例如按字段构造条件
- 最近数据库列表和欢迎页
- 更完整的诊断面板，而不是当前的摘要文本视图
