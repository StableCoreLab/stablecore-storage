# StableCore Storage 使用说明

这份文档面向“怎么用”的场景，目标是把当前仓库里已经落地的公共 API、基本流程和常见边界串起来，方便直接上手。

## 1. 先看哪里

建议先从这几个头文件入手：

- `Include/SCStorage.h`：一站式公共入口
- `Include/ISCInterfaces.h`：数据库、表、记录、事务接口
- `Include/SCTypes.h`：值类型、列定义、ChangeSet、Journal
- `Include/SCQuery.h`：查询计划与执行
- `Include/ISCComputed.h`：计算列、规则、缓存
- `Include/ISCTableView.h`：计算表视图
- `Include/SCBatch.h`：批处理与导入
- `Include/ISCDiagnostics.h`：诊断与变更集描述

## 2. 基本规则

使用这个库时，优先记住下面几条：

1. 写操作必须显式进入事务，入口是 `BeginEdit`，结束是 `Commit` 或 `Rollback`。
2. 读操作不应隐式写入。
3. 语义应与当前 SQLite 文件后端保持一致。
4. 删除后的记录句柄仍然可以被定位，但应通过 `IsDeleted()` 判断状态。
5. `SCStorage.h` 是推荐的统一包含入口。

## 3. 最小可用示例

下面是一个最小的内存数据库示例，演示建库、建表、建列、写入和提交。

```cpp
#include "SCStorage.h"

using namespace StableCore::Storage;

int main()
{
    SCDbPtr db;
    if (Failed(CreateFileDatabase(L"example.sqlite", SCOpenDatabaseOptions{}, db)))
    {
        return 1;
    }

    SCTablePtr table;
    if (Failed(db->CreateTable(L"Beam", table)))
    {
        return 1;
    }

    SCSchemaPtr schema;
    if (Failed(table->GetSchema(schema)))
    {
        return 1;
    }

    SCColumnDef width;
    width.name = L"Width";
    width.displayName = L"Width";
    width.valueKind = ValueKind::Int64;
    width.nullable = false;
    width.defaultValue = SCValue::FromInt64(0);
    if (Failed(schema->AddColumn(width)))
    {
        return 1;
    }

    SCEditPtr edit;
    if (Failed(db->BeginEdit(L"Create Beam", edit)))
    {
        return 1;
    }

    SCRecordPtr record;
    if (Failed(table->CreateRecord(record)))
    {
        return 1;
    }

    if (Failed(record->SetInt64(L"Width", 300)))
    {
        return 1;
    }

    if (Failed(db->Commit(edit.Get())))
    {
        return 1;
    }

    return 0;
}
```

## 4. 表、列、记录怎么用

### 4.1 建表

```cpp
SCTablePtr beamTable;
db->CreateTable(L"Beam", beamTable);
```

### 4.2 取 schema 并注册列

```cpp
SCSchemaPtr schema;
beamTable->GetSchema(schema);

SCColumnDef length;
length.name = L"Length";
length.displayName = L"Length";
length.valueKind = ValueKind::Double;
length.columnKind = ColumnKind::Fact;
length.nullable = false;
length.defaultValue = SCValue::FromDouble(0.0);
schema->AddColumn(length);
```

### 4.3 创建和读取记录

```cpp
SCRecordPtr record;
beamTable->CreateRecord(record);
record->SetDouble(L"Length", 6000.0);

double lengthValue = 0.0;
record->GetDouble(L"Length", &lengthValue);
```

### 4.4 删除记录

```cpp
beamTable->DeleteRecord(record->GetId());
```

删除后，`GetRecord(id, outRecord)` 仍然可能返回同一个逻辑记录句柄，但记录状态应通过 `IsDeleted()` 检查。

### 4.5 读取现有表内容

如果表已经存在，先通过 `GetTable` 取回表对象，再读取记录。

```cpp
SCTablePtr beamTable;
if (Failed(db->GetTable(L"Beam", beamTable)))
{
    return 1;
}
```

#### 遍历全部活记录

`EnumerateRecords` 只会返回活着的记录。

```cpp
SCRecordCursorPtr cursor;
if (Failed(beamTable->EnumerateRecords(cursor)))
{
    return 1;
}

SCRecordPtr current;
while (cursor->Next(current) == SC_OK && current)
{

    std::int64_t width = 0;
    if (current->GetInt64(L"Width", &width) == SC_OK)
    {
        // 使用 width
    }
}
```

#### 按记录 id 读取

如果你已经知道记录 id，可以直接按 id 读取。

```cpp
SCRecordPtr record;
if (Failed(beamTable->GetRecord(1, record)))
{
    return 1;
}
```

如果记录可能已经被删除，读取后要先检查：

```cpp
if (record->IsDeleted())
{
    // 已删除记录
}
```

## 5. 事务、Undo、Redo

所有写操作都应该放进事务里：

```cpp
SCEditPtr edit;
db->BeginEdit(L"Update Beam", edit);

SCRecordPtr record;
beamTable->GetRecord(1, record);
record->SetInt64(L"Width", 320);

db->Commit(edit.Get());
```

回滚：

```cpp
db->Rollback(edit.Get());
```

撤销和重做是数据库级行为：

```cpp
db->Undo();
db->Redo();
```

## 6. SQLite 数据库怎么打开

文件数据库通过 `CreateFileDatabase` 打开：

```cpp
SCDbPtr db;
SCOpenDatabaseOptions options;
options.openMode = SCDatabaseOpenMode::Normal;

if (Failed(CreateFileDatabase(L"example.db", options, db)))
{
    return 1;
}
```

常用打开模式：

- `Normal`：正常可读写
- `NoHistory`：不暴露历史能力
- `ReadOnly`：只读

### 6.1 备份

`CreateBackupCopy` 是文件后端能力，适用于 SQLite 持久化数据库。

```cpp
SCBackupOptions backupOptions;
backupOptions.preserveJournalHistory = true;
backupOptions.vacuumTarget = true;
backupOptions.validateTarget = true;
backupOptions.overwriteExisting = false;

SCBackupResult result;
db->CreateBackupCopy(L"backup.db", backupOptions, &result);
```

## 7. 查询怎么用

### 7.1 兼容路径：单条件等值查询

如果你只需要一个字段的等值匹配，最简单的是 `FindRecords`：

```cpp
SCQueryCondition condition;
condition.fieldName = L"Width";
condition.expectedValue = SCValue::FromInt64(300);

SCRecordCursorPtr cursor;
beamTable->FindRecords(condition, cursor);

SCRecordPtr current;
while (cursor->Next(current) == SC_OK && current)
{
}
```

这个路径只会枚举活着的记录。

### 7.2 计划化查询

查询层也支持 `QueryPlan`、`IQueryPlanner` 和 `ExecuteQueryPlan`。
适合你需要：

- 更复杂的条件组合
- 排序和分页
- 观察执行模式是直接命中、部分下推还是回退扫描

查询执行总入口遵循统一的 `QueryBackendKind` 分发，当前仅 SQLite 后端接入。

## 8. 计算字段（计算列）

计算字段（Computed Column）是一种只在内存中存在、不会被持久化到 SQLite 文件的虚拟列。它的值基于表达式、聚合函数或外部规则引擎动态计算得出，主要用于派生字段、跨表统计和业务规则注入。

### 8.1 三种计算方式

计算字段通过 `ComputedFieldKind` 决定计算方式：

| 方式 | 枚举值 | 说明 | 示例 |
|------|--------|------|------|
| 表达式 | `Expression` | 自研表达式引擎求值，支持算术运算、字段引用和内置函数 | `"Length * Width * Height"` |
| 规则 | `Rule` | 通过 `ISCRuleRegistry` 查找用户注册的 `ISCComputedEvaluator` | 外部注入的工程规范判定 |
| 聚合 | `Aggregate` | 跨表聚合，当前行通过关系链接到目标表，计算 Count/Sum/Min/Max | `Count(关联零件数)` |

### 8.2 表达式引擎

内置的表达式引擎是一个自研的词法/语法解析器，支持以下能力：

**算术运算：**
- `+` `-` `*` `/` 四则运算
- 一元负号 `-`
- 括号 `()` 控制优先级

**字段引用：**
- 直接写同表的字段名，如 `Length`、`Width`
- 求值时通过 `ISCComputedContext::GetValue()` 解析为数值

**内置函数：**

| 函数 | 说明 |
|------|------|
| `min(a, b)` | 取最小值 |
| `max(a, b)` | 取最大值 |
| `abs(x)` | 取绝对值 |
| `if(cond, then, else)` | 条件分支。cond 非 0 返回 then，否则返回 else |

**类型转换规则：**
- 所有参与运算的值先转为 `double`
- 表达式内部以 `double` 精度计算
- 最终结果按 `valueKind` 转回目标类型（Int64 四舍五入，Double 原样，Bool 非零为真）

**表达式示例：**

```cpp
// 体积计算
SCComputedColumnDef volume;
volume.expression = L"Length * Width * Height";

// 根据材质计算重量（混凝土 2400kg/m³，钢材 7850kg/m³）
SCComputedColumnDef weight;
volume.expression = L"if(isConcrete, volume * 2400, volume * 7850)";

// 取三边最大值
SCComputedColumnDef maxDim;
volume.expression = L"max(max(Length, Width), Height)";
```

### 8.3 聚合计算

聚合计算通过定义 `aggregateRelation`（格式为 `"表名.关系字段名"`）来关联目标表。支持四种聚合函数：

| 聚合函数 | 说明 |
|----------|------|
| `Count` | 统计关联记录数量 |
| `Sum` | 对 `aggregateField` 求和 |
| `Min` | 取 `aggregateField` 最小值 |
| `Max` | 取 `aggregateField` 最大值 |

**聚合示例：**

```cpp
SCComputedColumnDef screwCount;
screwCount.name = L"ScrewCount";
screwCount.displayName = L"螺丝数量";
screwCount.valueKind = ValueKind::Int64;
screwCount.kind = ComputedFieldKind::Aggregate;
screwCount.aggregateKind = SCAggregateKind::Count;
screwCount.aggregateRelation = L"Screw.ComponentRef";
// aggregateField 在 Count 模式下可不填
```

### 8.4 依赖声明与缓存

计算字段通过 `dependencies` 声明依赖哪些字段，这是缓存失效的关键依据：

```cpp
volume.dependencies.factFields = {
    {L"Beam", L"Length"},
    {L"Beam", L"Width"},
    {L"Beam", L"Height"},
};
```

- **factFields**：当前记录的字段依赖。格式为 `{表名, 字段名}`。
- **relationFields**：跨表关系依赖（影响聚合计算）。

当任一依赖字段发生变更时，相关缓存条目自动失效，下次读取时重新计算。

**缓存控制：**

```cpp
volume.cacheable = true;  // 默认开启。设为 false 则每次读取都重算
```

### 8.5 Rule 规则引擎

如果计算逻辑需要外部注入（例如工程规范、行业标准），可以使用 `Rule` 模式：

```cpp
SCComputedColumnDef grade;
grade.kind = ComputedFieldKind::Rule;
grade.ruleId = L"Steel.GradeChecker";  // 全局唯一规则 ID
```

使用前需要注册对应的求值器：

```cpp
// 实现 ISCComputedEvaluator
class SteelGradeChecker : public ISCComputedEvaluator
{
public:
    SCResult Evaluate(
        const SCComputedColumnDef& column,
        ISCComputedContext& context,
        SCValue& outValue) override
    {
        // 从 context 读取字段值
        SCValue strength, temp;
        context.GetValue(L"YieldStrength", &strength);
        context.GetValue(L"ServiceTemp", &temp);

        // 业务规则判断
        if (strength.ValueAsInt64() >= 345)
            outValue = SCValue::FromString(L"Q345");
        else
            outValue = SCValue::FromString(L"Q235");

        return SC_OK;
    }
};

// 注册
ruleRegistry->Register(L"Steel.GradeChecker",
    std::make_unique<SteelGradeChecker>());
```

### 8.6 完整 API 使用

**创建计算表视图：**

```cpp
#include "SCStorage.h"
using namespace StableCore::Storage;

SCComputedTableViewPtr view;
CreateComputedTableView(db.Get(), L"Beam", nullptr, view);
```

如果使用了规则引擎，第三个参数传入 `ISCRuleRegistry` 指针；如果只用表达式和聚合，传 `nullptr`。

**添加计算列：**

```cpp
SCComputedColumnDef volume;
volume.name = L"Volume";
volume.displayName = L"体积";
volume.valueKind = ValueKind::Double;
volume.kind = ComputedFieldKind::Expression;
volume.expression = L"Length * Width * Height";
volume.dependencies.factFields = {
    {L"Beam", L"Length"},
    {L"Beam", L"Width"},
    {L"Beam", L"Height"},
};
volume.cacheable = true;

view->AddComputedColumn(volume);
```

**读取计算值：**

```cpp
SCValue value;
view->GetCellValue(record->GetId(), L"Volume", &value);
double v = value.ValueAsDouble();
```

**遍历含计算列的表：**

```cpp
SCRecordCursorPtr cursor;
view->EnumerateRecords(cursor);

SCRecordPtr current;
while (cursor->Next(current) == SC_OK && current)
{
    SCValue vol;
    view->GetCellValue(current->GetId(), L"Volume", &vol);
    // 使用 vol
}
```

`EnumerateRecords` 返回的是底层事实表的游标，计算列的值按需读取。

**提取计算结果汇总：**
如果需要离线获取所有可提取的计算列汇总数据，可以使用 `ExtractComputedTableViewSummary`：

```cpp
SCComputedTableViewSummary summary;
if (Failed(ExtractComputedTableViewSummary(view.Get(), &summary)))
{
    // 处理失败
}
// summary.TableName, summary.ComputedColumns, summary.Records ...
```

**删除计算列：**

```cpp
view->RemoveComputedColumn(L"Volume");
```

### 8.7 编辑器中的使用

在数据库编辑器 UI 中，计算列的管理入口在菜单栏 **计算列(&C)** 和右侧检查器的 **计算列** 面板。

| 操作 | 菜单入口 | 说明 |
|------|---------|------|
| 添加会话计算列 | 计算列 → 添加会话计算列... | 弹出对话框，填写名称、类型、表达式/规则/聚合 |
| 编辑计算列 | 计算列 → 编辑选中的计算列... | 先在计算列面板选中，再编辑 |
| 转换为计算列 | 计算列 → 将选中列转换为计算列... | 将已有的事实列转为计算列（会清空该列数据） |
| 转换为事实列 | 计算列 → 将选中计算列转换为列... | 将计算列转回事实列 |
| 删除计算列 | 计算列 → 删除选中的计算列 | 移除计算列定义 |

**注意事项：**
- 计算列定义**不持久化**——关闭数据库后定义消失，下次打开需重新添加
- 计算列始终**只读**——不能在数据表格中编辑计算列的值
- 事实列转计算列时**会清空**该列所有已有数据
- 修改计算列定义后，视图和缓存会自动重建

### 8.8 缓存生命周期

```
事实字段变更
  → ISCDatabaseObserver::OnDatabaseChanged(changeset)
  → ISCComputedCache::Invalidate(changeset, computedColumns)
  → 遍历所有缓存条目
  → 若 changeSet 与条目的 dependencies 有交集 → 淘汰
  → 下次 GetCellValue 时重新计算
```

### 8.9 完整示例

```cpp
#include "SCStorage.h"
using namespace StableCore::Storage;

void SetupComputedColumns()
{
    // 1. 打开数据库
    SCDbPtr db;
    CreateFileDatabase(L"project.sqlite", SCOpenDatabaseOptions{}, db);

    // 2. 创建计算表视图
    SCComputedTableViewPtr view;
    CreateComputedTableView(db.Get(), L"Frame", nullptr, view);

    // 3. 添加体积计算列（表达式）
    SCComputedColumnDef volume;
    volume.name = L"Volume";
    volume.displayName = L"体积";
    volume.valueKind = ValueKind::Double;
    volume.kind = ComputedFieldKind::Expression;
    volume.expression = L"Length * Width * Height";
    volume.dependencies.factFields = {
        {L"Frame", L"Length"},
        {L"Frame", L"Width"},
        {L"Frame", L"Height"},
    };
    view->AddComputedColumn(volume);

    // 4. 添加螺丝数量统计列（聚合）
    SCComputedColumnDef screwCount;
    screwCount.name = L"ScrewCount";
    screwCount.displayName = L"螺丝数";
    screwCount.valueKind = ValueKind::Int64;
    screwCount.kind = ComputedFieldKind::Aggregate;
    screwCount.aggregateKind = SCAggregateKind::Count;
    screwCount.aggregateRelation = L"Screw.FrameRef";
    view->AddComputedColumn(screwCount);

    // 5. 读取
    SCRecordCursorPtr cursor;
    view->EnumerateRecords(cursor);

    SCRecordPtr record;
    while (cursor->Next(record) == SC_OK && record)
    {
        SCValue vol, count;
        view->GetCellValue(record->GetId(), L"Volume", &vol);
        view->GetCellValue(record->GetId(), L"ScrewCount", &count);
    }
}
```

## 9. 批处理和导入

批处理接口适合一次性写入多张表的数据。

```cpp
std::vector<SCBatchTableRequest> requests;

SCBatchTableRequest request;
request.tableName = L"Beam";
request.creates.push_back({
    {
        {L"Width", SCValue::FromInt64(300)},
        {L"Length", SCValue::FromDouble(6000.0)},
    }
});
requests.push_back(request);

SCBatchExecutionOptions options;
options.editName = L"Import Beams";
options.chunkSize = 128;
options.persistCheckpoints = true;
options.rollbackOnError = true;

SCBatchExecutionResult result;
ExecuteImport(db.Get(), requests, options, &result);
```

如果你要做可恢复导入，可以直接使用：

- `BeginImportSession`
- `AppendImportChunk`
- `GetImportRecoveryState`
- `FinalizeImportSession`
- `AbortImportSession`

SCStorageHealthReport report;

## 11. 常见返回码

几个最常见的错误码：

- `SC_OK`：成功
- `SC_E_POINTER`：输出参数为空
- `SC_E_INVALIDARG`：参数非法
- `SC_E_TABLE_NOT_FOUND`：表不存在
- `SC_E_COLUMN_NOT_FOUND`：列不存在
- `SC_E_RECORD_NOT_FOUND`：记录不存在
- `SC_E_RECORD_DELETED`：记录已删除
- `SC_E_NO_ACTIVE_EDIT`：没有活跃事务
- `SC_E_WRITE_CONFLICT`：事务冲突
- `SC_E_UNDO_STACK_EMPTY` / `SC_E_REDO_STACK_EMPTY`：栈为空

## 12. 推荐的调用顺序

一个比较稳妥的使用顺序是：

1. `CreateFileDatabase`
2. `CreateTable`
3. `GetSchema` 并注册列
4. `BeginEdit`
5. `CreateRecord` 或 `GetRecord`
6. `Set*` / `DeleteRecord`
7. `Commit` 或 `Rollback`
8. 必要时调用 `Undo` / `Redo`
