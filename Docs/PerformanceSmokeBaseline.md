# 性能冒烟基线

当前仓库已经包含基础级别的性能冒烟验证，主要位于：

- Tests/PerformanceSmokeTests.cpp

验证范围：

- 通过 ExecuteImport(...) 执行批量导入。
- 在持久化 SQLite 数据上执行索引等值查询。
- 通过关系列执行接近真实业务形态的查找。

本文强调的不是“严格性能指标”，而是“最低可接受能力基线”：

- 主路径不会因明显低效实现而失去可用性。
- 索引存在时能体现出基础收益。
- 批量导入与关系查询不会在小到中等数据量下出现明显失控。

当前结论：

- 仓库已经建立性能回归的最小护栏。
- 这些测试更适合发现明显退化，不适合替代正式基准测试。

后续建议：

- 增加不同数据规模的分层样本。
- 将性能结果纳入持续集成趋势跟踪。
- 对导入、查询、恢复分别建立更细的基线。
